## Overview

The Agent Assist API enables you to programmatically invoke SmartAgents to generate AI-powered responses for customer conversations. This endpoint is ideal for integrating XtendOps AI capabilities into your existing support platforms, chat applications, or custom workflows.

## Endpoint

```
POST https://agent-assist.xtendops.com/api/agent-assist
```

## Authentication

This endpoint requires authentication using a Bearer token. See the [Authentication](/api-reference/authentication) page for details on obtaining your API token.

## Request Headers

| Header          | Value              | Required | Description                   |
| --------------- | ------------------ | -------- | ----------------------------- |
| `Authorization` | `YOUR_API_TOKEN`   | Yes      | Your API authentication token |
| `Content-Type`  | `application/json` | Yes      | Must be `application/json`    |

## Request Body

### Required Fields

| Field            | Type   | Description                                     |
| ---------------- | ------ | ----------------------------------------------- |
| `aiAgentId`      | string | The ID of the SmartAgent to invoke              |
| `conversationId` | string | Unique identifier for the conversation thread   |
| `humanAgentId`   | string | Identifier for the human agent using the system |

### Optional Fields

| Field          | Type   | Default  | Description                                          |
| -------------- | ------ | -------- | ---------------------------------------------------- |
| `question`     | string | `""`     | The current customer question or message             |
| `responseType` | string | `"json"` | Response format (`json` or `text`)                   |
| `history`      | array  | `[]`     | Array of previous conversation messages              |
| `overrides`    | object | `{}`     | Custom metadata and inputs (accepts any JSON object) |

### Overrides Object

The `overrides` object is flexible and can accept any metadata as a JSON object. This allows you to pass custom context, variables, or data to your SmartAgent workflow.

**Common Use Cases:**

- Pass conversation history in a specific format
- Provide customer data (account info, order details, etc.)
- Include custom variables for workflow logic
- Override default SmartAgent inputs

## Response

### Success Response (200 OK)

```json
{
  "response": "Based on the order number you provided, I can see your order was placed on...",
  "conversationId": "conv_12345",
  "agentId": "6Qzpj3mJHrvxqstargemini",
  "metadata": {
    "processingTime": 1250,
    "model": "gpt-4"
  }
```

## Use Cases

### Customer Support Integration

Integrate AI assistance directly into your support platform:

```javascript
// Example: Zendesk integration
async function getAISuggestion(ticketData) {
  const response = await fetch(
    "https://agent-assist.xtendops.com/api/agent-assist",
    {
      method: "POST",
      headers: {
        Authorization: process.env.XO_API_TOKEN,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: ticketData.latestMessage,
        aiAgentId: "support-agent-id",
        conversationId: ticketData.ticketId,
        humanAgentId: ticketData.agentId,
        responseType: "json",
        overrides: {
          inputs: {
            chat_history: ticketData.conversationHistory,
            customer_tier: ticketData.customerTier,
            product: ticketData.product,
          },
        },
      }),
    }
  );

  return await response.json();
}
```

### Live Chat Automation

Provide real-time AI suggestions during live chat:

```javascript
// Example: Chat platform integration
async function getChatSuggestion(chatSession) {
  const response = await fetch(
    "https://agent-assist.xtendops.com/api/agent-assist",
    {
      method: "POST",
      headers: {
        Authorization: process.env.XO_API_TOKEN,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: chatSession.currentMessage,
        aiAgentId: "chat-agent-id",
        conversationId: chatSession.sessionId,
        humanAgentId: chatSession.agentId,
        history: chatSession.messages,
        overrides: {
          inputs: {
            customer_name: chatSession.customerName,
            page_url: chatSession.currentPage,
          },
        },
      }),
    }
  );

  return await response.json();
}
```

## Best Practices

1. **Use Unique Conversation IDs** - Generate unique IDs for each conversation thread to maintain context
2. **Provide Rich Context** - Use the `overrides` object to pass relevant customer data and metadata
3. **Handle Errors Gracefully** - Implement proper error handling for network issues and API errors
4. **Monitor Rate Limits** - Track your API usage to stay within rate limits (contact your XO teammate for details)
5. **Secure Your Token** - Never expose your API token in client-side code or public repositories
